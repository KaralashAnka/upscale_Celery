# Сервис для апскейлинга изображений

Этот проект представляет собой API для увеличения разрешения изображений с использованием модели EDSR (Enhanced Deep Residual Networks for Single Image Super-Resolution) на базе Flask, Celery и Redis.

## Особенности
- **Асинхронная обработка**: Тяжелые задачи по обработке изображений выполняются в фоновом режиме через Celery.
- **Оптимизация**: Модель загружается в память один раз и переиспользуется.
- **Работа в памяти**: Изображения не сохраняются на диск, обработка происходит в оперативной памяти (передаются как байты).
- **Docker**: Проект полностью контейнеризирован.

## Требования
- Docker
- Docker Compose

## Запуск
1. Склонируйте репозиторий.
2. Убедитесь, что файл модели `EDSR_x2.pb` существует`.
3. Запустите проект:
   ```bash
   docker-compose up --build
   ```

## API Эндпоинты

### 1. POST `/upscale`
Загрузка изображения для обработки.
- **Параметры**: `file` (form-data, файл изображения).
- **Ответ**: `{"task_id": "идентификатор_задачи"}` (Status Code: 202).

### 2. GET `/tasks/<task_id>`
Проверка статуса задачи.
- **Ответ**:
  - Пока задача выполняется: `{"status": "PENDING"}`.
  - Когда задача завершена: `{"status": "SUCCESS", "result": "http://localhost:5000/processed/<task_id>"}`.

### 3. GET `/processed/<task_id>`
Получение обработанного файла.
- **Ответ**: Файл изображения в формате PNG.

## Дополнительная информация
- Файлы передаются между Flask и Celery в виде байтов.
- Redis используется и как брокер сообщений, и как хранилище результатов (backend).
